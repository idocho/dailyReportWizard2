## 1. 프로젝트 개요

### 1.1 목적

DailyReportWizard 2.0이 Firebase에 축적한 일별 관찰 데이터와 성적 데이터를 기반으로,  
담임교사가 학부모에게 발송할 **월간 학생 성장 리포트**를 생성·검토·출력한다.

**핵심 가치**: 매일 쌓이던 데이터가 휘발되지 않고, 학부모에게 "우리 아이가 체계적으로 관리받고 있다"는 신뢰감을 주는 결과물로 변환된다.

### 1.2 구성 파일

| 파일 | 역할 | 런타임 |
|------|------|--------|
| `analyzer.html` | 단일 파일 — 생성·검토·출력 전체 | 브라우저 (웹) |

> `.py` 없음. 카카오톡 자동 전송이 목적이 아니며, API 키는 내부 도구 특성상 클라이언트에서 관리.

### 1.3 운용 시나리오

```
[월말] 담임교사가 웹브라우저에서 analyzer.html 접속
  1. 대상 반·기간 선택
  2. 일괄 생성 (AI가 전체 학생 리포트 순차 작성, ~2분)
  3. 학생별 검토 + 튜닝 (AI 서술 수정, 선생님 한마디 수정)
  4. 학생별 컨펌
  5. 컨펌된 학생 리포트 HTML 링크 생성 → 카카오톡 수동 전송
```

### 1.4 데이터 소스

```
Firebase (DRW 2.0과 공유, 읽기 전용)
├── config/sheets/      학생 명단·강사 정보
├── input/              과제수행도 + 특이사항
├── obs/                수업 관찰 태그 (v2.0 신규)
│     {sheet}|{cls}|{name}/{YYYY-MM-DD}
│       condition, understand, understand_sub[], engage[], caution[], extra[]
├── scores/             시험 유형별 점수 (v2.0 신규)
└── session/class_data/ 진도/과제
```

## 2. 화면 구성

### 2.1 전체 플로우

```
[Step 1] 생성 설정   →   [Step 2] 검토·튜닝   →   [Step 3] 출력
반·기간·항목 선택         학생별 검토·수정           HTML 링크 생성
일괄 생성 실행             컨펌                       (카카오 수동 전송)
```

### 2.2 Step 1 — 생성 설정

**대상 반 선택** (복수 선택 가능)
- 반 목록: Firebase `config/sheets/`에서 로드
- 각 반 카드: 반명 + 학생 수 표시

**기간 선택**
```
이번 달 / 지난 달 / 최근 3개월 (분기)
```

**포함 항목 선택** (기본 전체 체크)
```
☑ 역량 레이더 (5각형 스파이더 차트)
☑ 수업 컨디션 달력
☑ 관찰 집계 (긍정 태그 바)
☑ 성적 추이
☑ AI 서술 평가
☑ 기억에 남는 순간
☑ 다음달 집중 포인트
☑ 선생님 한마디
```

**실행 버튼**: `✨ N명 리포트 일괄 생성`

### 2.3 Step 2 — 검토·튜닝

```
┌──────────────┬──────────────────────────────────────┐
│  좌: 학생    │  우: 선택된 학생 리포트 검토 패널      │
│  목록        │                                       │
│              │  [학생명·기간 정보]  [재생성] [✓ 컨펌] │
│  김상덕 수정중│  ─────────────────────────────────── │
│  이민준 완료 │  섹션별 검토 영역                      │
│  박서연 완료 │  - AI 서술 (수정 가능)                 │
│  최지우 대기 │  - 자동 집계 (읽기 전용)               │
│  ...         │  - 선생님 한마디 (수정 권장)           │
│              │                                       │
│  [완료만 발송]│  [N명 완료] [발송 준비]               │
└──────────────┴──────────────────────────────────────┘
```

**학생 목록 상태**

| 상태 | 표시 | 색상 |
|---|---|---|
| 수정중 | 수정중 | 노랑 |
| 컨펌 완료 | 완료 | 초록 |
| 미검토 | 대기 | 회색 |

**검토 패널 섹션**

| 섹션 | 편집 | 설명 |
|---|---|---|
| AI 서술 평가 | ✅ | textarea 자유 수정 |
| 역량 레이더 | ❌ | 자동 집계, 읽기 전용 |
| 성적 요약 | ❌ | 자동 집계, 읽기 전용 |
| 컨디션 달력 | ❌ | 자동 집계, 읽기 전용 |
| 관찰 집계 | ❌ | 자동 집계, 읽기 전용 |
| 기억에 남는 순간 | ✅ | AI 선별, 수정 가능 |
| 다음달 집중 포인트 | ✅ | AI 생성, 수정 가능 |
| 선생님 한마디 | ✅ | AI 초안, **수정 권장** |

> **선생님 한마디**: 학부모에게 가장 직접 전달. AI 초안을 교사 언어로 다듬도록 UX 강조.

### 2.4 Step 3 — 출력

- 컨펌된 학생별 리포트 HTML 링크 생성
- 링크 목록 (학생명 + 링크) + 클립보드 복사 버튼

## 3. 학부모 리포트 HTML 구성

### 3.1 대상

- 모바일 세로 스크롤 기준 (카카오톡 인앱 브라우저)
- 최대 너비 380px

### 3.2 섹션 구성

```
[커버]
  학원명 배지 · 학생명 · 반/기간
  AI 커버 문장 (학생 특성 한 문장 요약)

[01] 선생님이 본 이번 달
  AI 서술 평가 (2~3문장)

[02] 역량 레이더 (5각형 스파이더 차트)
  5개 축 + 범례 바

[03] 한 달 수업 컨디션 (달력)
  날짜별 컨디션 색상
  번뜩임(금) / 잘함(파) / 보통(회) / 힘듦(분홍) / 미수업(흰)

[04] 이달에 보여준 것들
  긍정 관찰 태그 횟수 바 차트
  (발표 N회 / 혼자해결 N회 / 질문 N회 / 오답재풀이 N회)

[05] 성적 추이
  시험 유형별 최근 회차 바 차트
  학급 평균 대비 표시

[06] 선생님이 기억하는 순간
  AI 선별 하이라이트 2~3건 타임라인

[07] 이달의 키워드
  긍정 태그 + AI 분석 기반 성장 키워드

[08] 다음달 집중 포인트
  AI 생성 3가지

[푸터]
  선생님 한마디 (교사 수정 메시지)
  학원명 · 담임 · 발송일
```

### 3.3 5각형 역량 레이더 축

| 축 | 원천 | 산출 방식 | 범위 |
|---|---|---|---|
| ① 수업 태도 | `obs/{key}/{date}.condition` | great+good 비율 | 0~100 |
| ② 참여도 | `obs/{key}/{date}.engage` (발표+질문) | 발생 횟수 / 수업수 × 100 | 0~100 |
| ③ 과제 성실도 | `input/.../assign` | 성실 계열 / 수업수 × 100 | 0~100 |
| ④ 이해도 | `obs/{key}/{date}.understand` + `understand_sub` | 빠름 비율 + 긍정 태그 가중 | 0~100 |
| ⑤ 성취도 | `scores/` | 최근 시험 학급 내 백분율 | 0~100 |

### 3.4 주의 관찰 데이터 처리

`caution` 태그(졸음·폰·잡담·태도불량)는 학부모 리포트에 **직접 노출 금지**.  
AI가 "컨디션이 낮았던 날 집중력 저하가 함께 나타나는 경향이 있습니다" 등 완곡하게 표현하여 활용.

## 4. AI 생성 로직

### 4.1 API

- **모델**: `claude-sonnet-4-20250514` (Anthropic API)
- **호출 방식**: 학생별 데이터 JSON → 섹션별 텍스트 생성
- **일괄 생성**: 반 전체 순차 처리 (rate limit 고려, 병렬 지양)
- **API Key**: 클라이언트 로컬 보관 (내부 도구 특성)

### 4.2 생성 대상 섹션

| 섹션 | 프롬프트 입력 | 출력 |
|---|---|---|
| 커버 문장 | 월별 집계 + 메모 전체 | 1문장 |
| AI 서술 평가 | 전체 데이터 요약 | 2~3문장 |
| 기억에 남는 순간 | 메모 + 컨디션 기록 | 2~3건 선별 |
| 다음달 집중 포인트 | 약점 패턴 분석 | 3가지 |
| 선생님 한마디 | 전체 데이터 종합 | 1~2문장 |

### 4.3 AI 어조 원칙

- 학부모가 읽기 편한 따뜻하고 전문적인 톤
- 부정적 내용은 완곡하게, 성장 가능성 중심 표현
- `caution` 태그는 직접 언급 금지 — 완곡 표현만
- 학생 이름 직접 사용 ("상덕이는...")
- 어머님/아버님 호칭 사용 가능 (DRW 데일리 리포트와 다른 맥락)

## 5. Firebase 연동

### 5.1 읽기 전용

`analyzer.html`은 Firebase를 **읽기 전용**으로만 사용한다. 쓰기 없음.

### 5.2 읽는 노드

```
config/sheets/              학생 명단
input/                      과제수행도 + 특이사항
obs/                        수업 관찰 태그 (DRW 2.0 신규)
scores/                     시험 점수
session/class_data/         진도/과제
```

### 5.3 DRW 1.x 데이터만 있는 경우 처리

| 섹션 | 처리 |
|------|------|
| 역량 레이더 | 과제 성실도만 표시. 태도·참여도·이해도 축은 "데이터 없음" |
| 성적 추이 | 섹션 숨김 |
| 나머지 | 정상 표시 |
