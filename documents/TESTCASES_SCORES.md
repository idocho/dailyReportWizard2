## B. 저장 무결성 — 머지 / 키 이동 / 덮어쓰기 방지

| TC | 절차 | 기대 결과 |
|----|------|----------|
| B-1 **[Critical 회귀]** | 중1A 담임이 성취도평가 입력·저장 → 중1B 담임(같은 curriculum)이 같은 날짜·유형·회차로 신규 입력 → "병합" confirm 수락 → 저장 | ✅ DB `scores/achievement/{ck}/{testKey}/students`에 **A반+B반 점수 공존**. A반 점수 유실 없음 |
| B-2 | B-1 후 중1A 담임이 해당 시험 수정 → 자기 반 한 명 점수만 변경 → 저장 | ✅ B반 점수 보존, A반 변경분만 반영 (서브경로 PATCH — `meta`·렌더 키만 전송) |
| B-3 | 주간Test 저장 → 학생 1명 타 반으로 이동(명단 변경) → 해당 시험 수정·저장 | ✅ 전출생 점수 students 맵에 보존(렌더 안 된 키 미삭제) |
| B-4 | 시험 수정에서 특정 학생 점수 지우고(빈칸) 저장 | ✅ 그 학생 키만 DB에서 삭제(null PATCH). 다른 학생 무영향 |
| B-5 | 신규 작성 시 기존과 같은 날짜·유형·회차 입력 → 저장 | ✅ "기존 기록에 이어서(병합) 저장할까요?" confirm. 취소 시 저장 안 됨 |
| B-6 | 기존 주간Test(반별) 수정 → 날짜 변경 → 저장 | ✅ 구 testKey 노드 삭제 + 새 `날짜\|유형` 키로 이동. 목록에 1건만(고스트 없음). ※ 반별 시험만 날짜가 키에 포함 |
| B-7 | 기존 성취도평가(achievement) 수정 → 유형을 주간Test로 변경 → 저장 | ✅ `scores/achievement/`에서 삭제 + `scores/weekly/{classId}/{subject}/`에 생성. 목록 중복 없음 |
| B-8 | B-6에서 이동 대상 키에 이미 다른 기록 존재 | ✅ "그 기록과 병합할까요?" confirm 후 병합 |
| B-9 | 직접입력 시험명에 `중간/기말 1.5차 [A]` 입력 → 저장 | ✅ 저장 성공. testKey는 금지문자 치환(`중간-기말 1-5차 -A-`), **meta.type은 원형 보존** → 목록·발송 표기 원형 |
| B-10 | 회차에 `1|2` 입력 → 저장 | ✅ 키 구분자 충돌 없음(치환), meta.round 원형 |
| B-11 | **기출모의고사** 신규 저장 (v8.20 단위 정정) | ✅ `scores/weekly/{classId}/{subject}/` 저장(반별). 「반」 배지. 담당 강사 권한으로 입력 가능(담임 불요) |
| B-12 | (잔존 데이터) `scores/achievement/`에 남은 구 기출모의고사 기록 | ✅ 목록에 「학년」 배지로 표시·삭제 정상(kind 기준). **수정·저장 시 weekly로 자동 이동**(achievement에서 삭제, 고스트 없음) |
| B-13 **(v8.23)** | 3MGM 담임이 실전모의고사 1회를 6/12 날짜로 저장 → 3MSM 담임이 같은 시험을 6/13 날짜(또는 기본 오늘 날짜 방치)로 입력 | ✅ **같은 시험으로 식별**(키=`실전모의고사\|1`, 날짜 무관) → 병합 confirm 없이 제자리 머지(수정) 또는 신규 시 병합 confirm. 학년 집계에 두 반 합산 |
| B-14 **(v8.23)** | 학년 시험 회차를 한 강사는 `1`, 다른 강사는 `1회`로 입력 | ✅ 키 정규화로 같은 시험 식별(`\|1`). meta.round 표시는 각자 입력 원형(마지막 저장 기준) |
| B-15 **(v8.23)** | 8.23 이전 날짜 포함 구키(`2026-06-12\|성취도평가`)로 저장된 학년 기록 수정 → 저장 | ✅ 새 키(`성취도평가`)로 자동 이동, 구 노드 삭제. 목록 1건 유지 |
| B-16 **(v8.23)** | 같은 과정에서 회차 없이 두 번째 실전모의고사 신규 작성 | ✅ "같은 유형·회차의 기록이 이미 있습니다" confirm — 다른 시험이면 취소 후 **회차 입력으로 구분**(학년 시험 회차 권장) |

## C. 입력 검증 / 표시

| TC | 절차 | 기대 결과 |
|----|------|----------|
| C-1 | 만점 100, 점수 150 입력 → 저장 | ✅ "0~100점 범위를 벗어난 점수가 1건" toast, 저장 차단 |
| C-2 | 점수 -5 입력 → 저장 | ✅ 동일 차단 |
| C-3 | 점수 0 입력 → 저장 | ✅ 정상 저장(0점은 유효) |
| C-4 | 만점 입력란 95로 변경 | ✅ 행 `/95` 갱신, **테이블 헤더 "만점" 라벨은 그대로**(회귀: 종전 "/95"로 변함), input `max=95`, 통계 갱신 |
| C-5 | 점수 입력 중 실시간 통계 | ✅ 평균·최고·최저·인원 즉시 갱신 |
| C-6 | 목록 정렬: 날짜 다른 시험 여러 개 + B-6처럼 키-날짜 불일치였던 기록 | ✅ `meta.date` 최신순 정렬 |
| C-7 | 학년 집계 줄(🎓): 만점 150 성취도평가 | ✅ 평균/최고/최저 **원점수 그대로**(점 단위, v8.21). 예: 평균 120.0점 · 최고 135점 · 최저 98점 · N명 |
| C-8 | 타 기기/강사가 시험 삭제 → 내 탭에서 성적 입력 재진입 | ✅ 빈 노드(null) 응답 시 스테일 목록 클리어. 네트워크 실패 시엔 기존 로컬 유지 |
| C-9 | 학년 시험에 두 반 학생 입력(예: 3MGM 80점, 3MSM 90점) → 각 반에서 목록 확인 (v8.22) | ✅ 카드 "반 평균" = **자기 반 학생만**(3MGM에서 80점, 3MSM에서 90점), 입력 인원도 반 기준 n/N. 🎓 학년 집계 = 두 반 합산(평균 85.0점 · 2명) |
| C-10 | C-9 상태에서 시험 수정 화면 진입 (v8.22) | ✅ 하단 통계 초기값부터 현재 반 학생만 집계(첫 입력 시 값 점프 없음) |

| D-5 | 마지막 남은 반의 점수를 담임이 삭제 | ✅ 빈 노드 자동 정리(테스트 키 삭제), 휴지통 백업 존재 |
| D-6 | **관리자**가 학년 시험 삭제 | ✅ "[관리자] 모든 반의 점수가 삭제됩니다" confirm → 전체 노드 삭제 + 휴지통 백업 |
| D-7 | 휴지통 백업 실패 상황(네트워크 차단 등) | ✅ **삭제 자체가 중단**(fbFail toast). 백업 없는 파괴 없음 |
| D-8 | 관리자 → 성적 탭 하단 🗑 휴지통 열기 → 복원 | ✅ 원경로에 전체 노드 복원(같은 키 존재 시 덮어씀 confirm), 휴지통에서 제거, 목록 재로드 |
| D-9 | 휴지통 영구 삭제 | ✅ "복원할 수 없습니다" confirm 후 trash 항목 제거 |
| D-10 | 비관리자 화면 | ✅ 🗑 휴지통 버튼 미노출(adminOn 전용) |

## G. 회귀 스모크 (전 변경 공통)

1. 신규 시험 추가 → 저장 → 목록 표시 → 수정 → 저장 → 삭제 전 과정 에러 없음 (콘솔 무에러).
2. 성적 입력 탭 ↔ 수업 입력 탭 전환, 담당 수업 전환 시 목록 정상 리로드.
3. 권한: 미담당 수업 weekly 저장 차단 toast / 비담임 achievement 저장 차단 toast.
4. 캐시버스트: 강력 새로고침 없이 `?v=202606121100` 로드 확인 (Network 탭).
