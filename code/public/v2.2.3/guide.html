DailyReportWizard는 두 부분으로 이루어집니다. ① 웹 앱 — 태블릿/PC 브라우저에서 매일의 수업·성적을 입력합니다. ② PC 전송 클라이언트 — 입력된 내용을 받아 AI 특이사항을 만들고 카카오톡으로 전송합니다. 먼저 1~2부에서 웹 앱을 설정하고, 전송 담당자는 3부에서 PC 클라이언트를 내려받아 설치하세요.

🔑
먼저 알아둘 2가지 고정값. Firebase 연결 단계(4단계)에서 아래 두 값을 그대로 입력하면 됩니다. 베타 기간 동안 변경되지 않습니다.
DB URLhttps://dailyreportwizard-default-rtdb.firebaseio.com
경로drw2_cbt

1부 · 웹 앱 접속

1

준비물 확인

  • PC 또는 태블릿 — 데스크톱/노트북, 또는 9인치급 이상 태블릿(가로 모드 권장).
  • 브라우저 — Chrome 또는 Edge 최신 버전 권장 (Safari도 동작).
  • 인터넷 연결 — 데이터는 클라우드(Firebase)에 저장되므로 접속이 필요합니다.
2

앱 주소로 접속

브라우저 주소창에 아래 주소를 입력합니다.

앱 주소https://dailyreportwizard.web.app

로그인 화면이 아니라 곧바로 앱 화면(왼쪽 사이드바 + 본문)이 뜨면 정상입니다.

2부 · 최초 1회 설정 (설정 위저드)

처음 접속하면 화면 가운데에 설정 위저드가 자동으로 나타나 4단계로 안내합니다. 순서대로 따라가기만 하면 됩니다. 설정은 이 기기에 저장되어, 다음부터는 위저드 없이 바로 사용 화면이 뜹니다.

💡
위저드 아래 나중에 할게요 를 누르면 건너뛰고 설정 화면에서 직접 입력할 수도 있습니다. 강사 계정을 만들기 전까지는 다음 접속 때 위저드가 다시 나타납니다.
1

위저드 1단계 · 🔥 Firebase 연결

데이터를 저장·동기화할 주소를 입력하는 화면입니다. 아래 두 고정값을 그대로 넣으세요.

DB URLhttps://dailyreportwizard-default-rtdb.firebaseio.com
경로 (Secret Path)drw2_cbt

💾 연결 정보 저장 클릭 → 다음 → 을 누릅니다.

⚠️
두 값의 철자·대소문자가 정확해야 합니다. 복사 버튼을 쓰면 오타를 막을 수 있습니다. 머리(https://)나 끝 슬래시(/)를 임의로 더하거나 빼지 마세요.
2

위저드 2단계 · 🔑 내 계정

입력칸에 본인 이름을 적고 조회 를 누릅니다.

  • 이미 등록된 이름이면 → 바로 그 계정을 불러옵니다.
  • 없는 이름이면 → 자동으로 신규 등록되며 "✓ ○○○ 계정 준비 완료"가 표시됩니다.

완료되면 다음 → 으로 진행합니다.

💡
입력한 이름이 그대로 계정 키가 됩니다. 동료와 표기를 통일하세요(예: "홍길동" vs "홍길동T"). 담당 수업·학생 명단은 웹에서 구성하며, PC 앱은 이 계정으로 데이터를 받아옵니다.
3

위저드 3단계 · 👥 학생 명단 불러오기

📥 학생 명단 불러오기 버튼을 누릅니다. 연결된 Firebase에서 학급·학생·과목 데이터를 이 기기로 내려받습니다.

"✓ ○개 학급 · 학생 ○명 불러옴"이 표시되면 성공입니다. 다음 → 으로 진행합니다.

ℹ️
명단이 비어 있어도 정상입니다(신규 학원). 다음 단계에서 과목·교재를 직접 만들 수 있습니다.
4

위저드 4단계 · 📚 담당 수업 배정

내가 가르치는 수업을 추가합니다. 반 선택 → 과목 선택 → 역할(담임/부담임) 선택 → + 수업 추가.

📘
교재가 아직 없을 때. 선택한 반에 등록된 과목(교재)이 없으면 “⚠️ 이 반에 등록된 교재 없음” 과 함께 과목 등록 폼이 나타납니다. 과정(예: 중3-1)을 고르고 교재명(예: 개념원리)을 입력해 ✓ 를 누르면 과목이 즉시 만들어지고 곧바로 배정할 수 있습니다.

다 됐으면 완료 → 를 누릅니다.

ℹ️
반 자체가 없다면 이전 단계에서 학생 명단을 먼저 불러오세요. 학급 생성은 🏫 학급 & 학생 관리 에서 합니다(운용 가이드 참고).
✓

완료 — 🚀 시작하기

마지막 화면에 Firebase 연결 / 강사 계정 / 학급·학생 수 / 담당 수업 수 요약 체크리스트가 표시됩니다. 🚀 시작하기 를 누르면 위저드가 닫히고 수업 입력 화면으로 들어갑니다.

담당 반의 학생 목록이 보이면 웹 앱 준비 끝입니다. 매일 사용법은 운용 가이드 탭을 참고하세요. 카카오톡 전송을 담당한다면 아래 3부로 이어집니다.

3부 · PC 전송 클라이언트 (카카오톡 발송)

웹 앱에 입력된 수업·성적 데이터를 받아 AI 특이사항을 생성하고 학부모에게 카카오톡으로 일괄 전송하는 Windows 전용 프로그램입니다. 전송을 담당하는 PC에만 설치하면 됩니다.

💻
설치 조건. Windows 10/11 PC + 카카오톡 PC 버전이 설치·로그인되어 있어야 합니다. 전송은 카카오톡 PC 창을 자동 제어하는 방식이라, 실행 중 PC를 다른 작업에 쓰지 마세요.
1

프로그램 다운로드

아래 버튼으로 최신 실행 파일을 내려받습니다. (약 23MB, 설치 과정 없는 단일 실행 파일)

⬇ DailyReportWizard_v2.2.2.exe 다운로드 최신 버전 v2.2.2

전체 릴리즈 목록: github.com/idocho/dailyReportWizard2/releases

2

실행 & 보안 경고 통과

받은 DailyReportWizard.exe 를 더블클릭합니다.

서명되지 않은 프로그램이라 "Windows의 PC 보호" 파란 창이 뜰 수 있습니다 → 추가 정보 클릭 → 실행 을 누르면 정상 실행됩니다.

⚠️
백신이 차단하면 허용/신뢰 처리하세요. 사내 배포용 자체 제작 프로그램이라 안전합니다.
3

최초 설치 위저드 (3단계)

처음 실행하면 메인 창에 설치 위저드가 나타나 3단계로 안내합니다(별도 팝업 아님). 완료하면 그 자리에서 정상 화면으로 전환됩니다.

① 🔥 Firebase 연결 — DB URL과 경로를 입력하고 ⚡ 연결 테스트 로 확인합니다. 웹 앱과 같은 값입니다.

DB URLhttps://dailyreportwizard-default-rtdb.firebaseio.com
경로drw2_cbt

② 🔑 내 계정 — 웹에서 만든 본인 이름을 입력하고 조회 및 설정 을 누릅니다(없으면 자동 등록). 담당 수업이 함께 동기화됩니다.

③ 🤖 AI 엔진 · API 키 — 특이사항 자동 생성에 쓸 엔진을 고르고 키를 입력합니다. 화면에 엔진별 발급 방법이 바로 안내됩니다(아래 4부 참고). 지금 안 쓰면 건너뛰기 가능.

마지막 요약에서 ✅ 설정 완료 를 누르면 끝납니다.

💡
같은 폴더에 설정 파일이 자동 생성됩니다. exe를 고정 폴더(예: 바탕화면의 전용 폴더)에 두고 쓰세요. 학생 명단·담당 수업은 웹에서 구성하고, PC는 다음 단계의 📥 데이터 가져오기로 받아옵니다.
ℹ️
설정을 나중에 바꾸려면 상단 ⚙️ 설정 에서 동일 항목을 수정할 수 있습니다.
4

데이터 가져오기 → AI 생성 → 전송

  • 📥 데이터 가져오기 — 웹 앱에 입력된 오늘 수업·성적·관찰 태그를 불러옵니다.
  • ✨ AI생성 — 학생별 특이사항 문장을 자동 작성합니다(단건/일괄).
  • 🚀 전송 — 버튼을 누르면 전송 대상 선택 창이 뜹니다. 준비완료 학생이 모두 체크된 상태이며, 보내지 않을 학생만 체크를 해제([전체 선택]/[전체 해제] 버튼 제공)한 뒤 전송 시작을 누르면 카카오톡 학부모 채팅방으로 차례대로 발송됩니다.
🆕
v2.1.2 — 선택 관찰 태그 한눈에. 가져오기 후 중앙 패널의 교재별 수행도 아래에 그날 웹에서 고른 관찰 태그가 한 줄로 표시됩니다(주의=빨강·하이라이트=초록). AI생성·검토 시 참고하세요.
🗂
v2.1.2 — 전송 코멘트 누적. 전송된 최종 특이사항은 날짜별로 자동 보관되어, 월간 리포트(Analyzer)에서 지난 코멘트와 중복되지 않게 활용됩니다. 별도 조작은 없습니다.
⏹
전송 중 취소 — 전송이 시작되면 버튼이 "⏹ 전송 취소"로 바뀝니다. 누르면 현재 학생까지만 보내고 멈추며, 아직 보내지 않은 학생 데이터는 그대로 유지되어 다시 전송할 수 있습니다.
🖱️
전송 중에는 마우스·키보드를 건드리지 마세요. 카카오톡 창을 자동 조작하는 동안 방해되면 잘못된 방에 전송될 수 있습니다.

대상 반 선택

담당 수업이 여러 개면 상단 탭(또는 그룹 탭)에서 입력할 반·과목을 고릅니다. 화면 제목에 반 · 학년학기 · 과목과 오늘 날짜가 표시됩니다.

🔧
제목 우측의 날짜 변경 버튼은 테스트용으로 다른 날짜의 데이터를 입력/확인할 때 씁니다. 평소엔 건드리지 않아도 됩니다.
2

진도 입력

상단 진도 칸에서 대단원 → 소단원 순으로 선택합니다(커리큘럼이 연결돼 있으면 목록이 자동으로 뜸). 단원 목록이 없거나 직접 적고 싶으면 ✏️ 직접 입력 을 고르세요. 아래 미리보기로 최종 표기를 확인합니다.

3

과제 입력

과제 칸에서 형식을 고릅니다: p. 페이지 / # 번호 / 직접 입력. 페이지·번호는 시작~끝을 적으면 p.12~p.20 처럼 자동 정리됩니다.

💡
진도·과제는 반 전체 공통이라 한 번만 입력하면 됩니다. 입력 즉시 자동 저장됩니다.
4

학생별 평가 입력

아래 👥 학생별 입력에서 학생마다 카드 한 장씩 채웁니다. 각 줄의 의미:

항목입력 방식
과제수행 등급 1개 선택(택1) + 필요 시 자주 쓰는 문구(복수 선택).
컨디션상태 1개 선택(화살표 아이콘).
이해도이해 수준 1개 + 세부 태그(복수) + 하이라이트.
참여수업 참여 태그(복수 선택).
기타특이/주의 태그(복수 선택).
메모자유 서술 특이사항(한 줄).

모든 입력은 누를 때마다 즉시 저장됩니다. 별도 저장 버튼이 없습니다.

5

완료 표시(점) 읽기

학생 이름 옆 점과 상단 n/n명 완료 카운터로 진행 상황을 봅니다.

과제 평가 + 진도/과제 입력 완료
과제 평가는 했으나 진도/과제 미입력
아직 과제 평가 안 함

모든 학생이 초록색이 되도록 채우는 것이 목표입니다.

② 성적 입력

사이드바 📊 성적 입력 탭. 좌측에서 반·과목을 고른 뒤 시험을 추가하고 원점수를 입력합니다.

1

새 시험 추가

+ 새 시험 추가 클릭 → 시험 유형, 회차(선택), 날짜, 만점, 메모(시험 범위 등)를 설정합니다.

2

점수 입력 & 저장

학생별 점수 칸에 원점수를 입력합니다. 평균 등 통계가 자동 계산됩니다. 입력 후 저장 하세요.

⚠️
입력 권한. 주간/단원 시험은 해당 과목 담당 강사만, 학년단위 시험은 담임만 입력할 수 있습니다. 권한이 없으면 저장 시 안내 토스트가 뜹니다.

문제 해결 (FAQ)

화면이 비어 있거나 학생이 안 보여요

① ⚙️ 설정에서 Firebase 연결이 연결됨 ✓ 인지 확인 → 아니면 4단계 재진행. ② 📥 학생 명단 불러오기 클릭. ③ 담당 수업이 등록돼 있는지 확인(7단계).

"연결 실패" 토스트가 떠요

DB URL 또는 경로 오타일 가능성이 큽니다. 설치 가이드 상단의 복사 버튼으로 두 값을 다시 붙여넣고 저장하세요. 인터넷 연결도 확인하세요.

입력했는데 다른 기기에서 안 보여요

정상입니다. 다른 기기에서 📥 학생 명단 불러오기 로 수동 갱신하면 보입니다(위 '데이터 동기화 주의' 참고).

성적 저장이 안 돼요

입력 권한 문제일 수 있습니다. 주간/단원 시험은 과목 담당, 학년단위 시험은 담임만 저장 가능합니다. 본인 역할(담임/부담임)을 📚 내 담당 수업 에서 확인하세요.

삭제한 교재가 PC 앱에 계속 보여요

웹과 PC 모두 v2.2.2 이상이어야 합니다. v2.2.2부터 과목 삭제가 '보관' 방식으로 바뀌어 모든 기기에서 일관되게 숨겨집니다. 구버전 PC 앱은 보관 과목을 인식하지 못하니 최신 exe로 업데이트하세요. 이미 보관한 과목을 되살리려면 학급 관리에서 같은 과정·교재를 다시 추가하면 됩니다.

화면이 작다고 차단되나요?

베타 기간에는 소형 화면 차단을 해제해 두었습니다. 다만 9인치급 이상 태블릿(가로) 또는 PC가 가장 편합니다. 좁으면 브라우저 배율을 줄여보세요.
